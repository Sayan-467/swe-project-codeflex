🔗 HOW TO USE THE API
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Endpoint: GET /fetch/editorial
  Parameter: problem_url=<codeforces_problem_url>

  Example URLs:
    • http://127.0.0.1:8000/fetch/editorial?problem_url=https://codeforces.com/contest/1741/problem/B
    • http://127.0.0.1:8000/fetch/editorial?problem_url=https://codeforces.com/problemset/problem/2149/A

⚙️ WHAT EACH FILE DOES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  main.py
    - FastAPI application
    - Defines /fetch/editorial endpoint
    - Handles HTTP requests/responses

  cf_editorial.py
    - Selenium web scraping (bypasses 403 blocks)
    - Finds tutorial links from problem pages
    - Fetches and cleans editorial text
    - Codeforces API authentication

  requirements.txt
    - List of Python packages needed
    - Use: pip install -r requirements.txt

  .env
    - Contains API credentials
    - NEVER commit this to Git!
    - Format:
        CODEFORCES_API_KEY=your_key
        CODEFORCES_API_SECRET=your_secret

  start_server.ps1
    - Automated startup script
    - Activates venv and starts server

  test_preview.py
    - Quick test to verify everything works
    - Shows formatted editorial preview


💡 TIPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ✓ Use Ctrl+C to stop the server
  ✓ Check http://127.0.0.1:8000/docs for interactive API testing

  ✓ Editorial text is automatically cleaned for readability
